Descripción

Whub es una plataforma enfocada al desarrollo colaborativo, abierto y libre de sitios web. Para ello se replica el modelo de desarrollo colaborativo de software, además de una interfaz web que permite abstraer los aspectos técnicos y promover la participación de personas sin conocimientos en el área de desarrollo. El flujo de trabajo es el siguiente:

  • Una persona crea un proyecto de página web, el cuál será replicado y modificado por todos los colaboradores. Quien crea el proyecto en primera instancia es el autor.

  • Una vez creado el proyecto, el autor puede invitar a más personas a pertenecer al grupo de autores. Todos los que pertenezcan a este grupo podrán aprobar los aportes de los demás colaboradores.

  • Cualquier persona puede aportar al proyecto en tres simples pasos:
    1. Crear copia local del proyecto.
    2. Realizar cambios (interfaz para escribir documentos).
    3. Enviar cambios (sujetos a aprobación).

  • Cuando un colaborador envía cambios los autores son notificados. Cualquier autor puede ver los cambios propuestos y aprobarlos/rechazarlos, incluyendo conflictos (solapamiento) entre distintos aportes.

La principal diferencia de Whub en relación con el resto de herramientas para crear sitios web es el paradigma bajo el cual funciona, el paradigma peer-to-peer (p2p) de producción colaborativa.

Aspectos técnicos

Para el desarrollo de Whub se utilizará PHP en el back-end y Bootstrap (v3) en el front-end. Como soporte para la producción colaborativa se utiliza GitHub, específicamente la plataforma GitHub Pages. Se espera que un futuro se pueda utilizar Whub con cualquier servidor que corra git, incluso servidores propios, de manera de crear nodos autónomos de producción de sitios. Gracias al soporte proporcionado por GitHub Pages será posible imitar el comportamiento de un blog, con posts categorizados por fecha y autor, sin necesidad de usar base de datos.

  • Conocimiento de la API de GitHub.
